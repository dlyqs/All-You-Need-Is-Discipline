# 判断卖出 Skill

## 目标

判断当前持仓标的是否应该卖出、减仓、继续持有或观察。这个 skill 必须结合买点质量、趋势是否破位、题材是否被资金抛弃、用户持仓成本和利润垫。

## 输入要求

- `symbol_or_name`: 当前持仓标的。
- `quote_snapshot`: 行情脚本输出的近期基础行情和当日粗略走势。
- `user_profile`: 用户风险偏好、资金纪律。
- `portfolio_holding`: 当前数量、买入时间、买入价、成本、浮盈浮亏、持仓逻辑、止损/止盈备注。
- `model_market_research`: 模型临时查询得到的大盘、板块、标的逻辑、风险事件、K 线形态。

## 数据使用边界

- 行情脚本用于确认近期涨跌、当日涨跌、粗略走势、换手等事实。
- 趋势是否破位、是否高位上影、是否反包、题材是否被抛弃、行业平均表现、黑天鹅影响，由模型临时查询和看图分析。
- 如果没有持仓信息，必须列为缺失证据，不能给确定卖出结论。
- 如果缺少 `quantity`，或缺少 `buy_date`，或缺少 `buy_price`/可替代成本信息，必须先要求用户补充，不能进入卖出判断。

## 不卖规则

逐条判断并说明：

1. 还在走趋势，没有跌破位，有利润垫，且股性好的票不用急着卖。
2. 开盘一字，或早盘一字后续封住，或极特别开了但图形完全还是一字的不用急着卖。
3. 是主线轮动题材，股性较好，且在平台震荡期刚买入一两天但只有小几个点亏损的不用急着卖。
4. 标的本身没问题，但遇到黑天鹅事件，且已经错过前期止损点的，不用在谷底卖。

## 必须卖规则

逐条判断并说明：

1. 买点极差，例如高位上影线且次日还绿没反包，需要及时止损。
2. 原本是热点股票且股性很好，但最近逐渐被资金抛弃，跑不赢行业平均，反弹时需要止损清仓。
3. 对该股票没有继续长期上涨逻辑，前期已有夸张上涨异动且风险很大，同时已经赚了 10% 或 20%，需要清仓止盈。

## 加分项

卖出判断里的加分项用于支持继续持有或减仓而不是立刻清仓：

1. 仍然有清晰中长期上涨逻辑。
2. 板块仍在主线或强轮动里。
3. 当前亏损主要来自系统性事件，而不是标的逻辑破坏。
4. 有利润垫且趋势未破位。

## 评级规则

- `must_sell`: 明确命中必须卖规则，或买点/逻辑/走势同时恶化。
- `reduce`: 有风险但不必清仓，适合反弹减仓、降低仓位或释放现金。
- `hold`: 不卖规则明确成立，趋势和逻辑仍可支撑持有。
- `watch`: 证据不足或需要等关键价位/分时/板块确认。

## 输出结构

必须输出：

- `rating`
- `conclusion`
- `rule_matches`: 对“不卖规则”和“必须卖规则”逐条说明。
- `bonus_matches`: 对继续持有支持项逐条说明。
- `vetoes`: 这里记录命中的必须卖规则。
- `missing_evidence`
- `action`: 清仓/减仓/继续拿/观察，以及触发价位或条件。

## 输出要求

- 必须结合用户买入价、买入时间、亏损或盈利幅度。
- 如果买入价、买入时间、数量或持仓逻辑不完整，先输出补充问题，不要直接给 `must_sell/reduce/hold`。
- 必须说明当前卖出建议是因为买点差、趋势破位、题材被抛弃、止盈、还是现金效率。
- 不能只看当日涨跌就给卖出结论。
